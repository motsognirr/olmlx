name: explain
description: Explain code or concepts clearly, adapting depth to the question
---

When explaining code or technical concepts:

1. **Start with the "what"** — one sentence summarizing what this code/concept does
2. **Then the "why"** — what problem it solves or why it exists
3. **Then the "how"** — walk through the mechanism step by step

## Adapting depth
- For simple questions, keep it brief — don't over-explain
- For complex topics, use analogies and build up from fundamentals
- If explaining code, trace through a concrete example with real values

## Formatting
- Use short paragraphs, not walls of text
- Use code snippets to illustrate points
- Bold key terms on first use

Avoid jargon unless the user used it first. If you must use a technical term, define it inline.
